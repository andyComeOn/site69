数据库表的操作指令

指令名称 指令详情 指令名称 指令详情
查看所有表(前提已经切换具体的某个数据库) show tables; 创建表 create table `表name` (字段1 数据类型, 字段2 数据类型, 字段3 数据类型);
字符集包括 utf, gbk, gb2312, big5; 表类型包括 nnoDB, MyIsam, BDB 默认是InnoDB
查看表结构 desc `name`; 查看表的创建语句 show create table `name`;
删除表 drop table `name`; 表-添加字段 alter table `name` add 字段名 字段类型 [字段属性...(eg: default 0)] [after 某字段名 或first];
表-修改字段 alter table `name` change 旧字段名 新字段名 新字段类型 [字段属性...]; 表-修改字段(仅修改字段的其他信息，则可以使用) alter table `name` modify 要修改字段名 字段类型 [字段属性...];
表-删除字段 alter table `name` drop 字段名; 表-修改表名 alter table `name` rename 新表名;
表-修改其字符集 alter table `name` charset=新字符集;

数据库-数据操作初步

指令名称 指令详情 指令名称 指令详情
数据基本4种操作 增(insert), 删(delete),改(update), 查(select); 数据基本4种操作，也称：CRUD操作 Create(创建), Retrieve(获取), Update(更新), Delete(删除);
插入数据 insert into 表名 (字段名1， 字段名2， ... ) values (数据1， 数据2， ... ); *字段名和数据是“一一对应”，对要写入的数据str、时间日期类型，用单引号引住 可省略“字段列表”部分，但需给出跟字段数量一样多的数据(通俗说就是位数对齐) eg: insert into 表名 (id) values (1) 都是1位
删除数据 delete from 表名 [where 条件]; *删除数据指的是删除表的某些行，比如原来有10行，可以将其中的3行删除，则剩下7行 where条件表示删除数据所应满足的条件，含义跟select一样。注：where 条件可不写，若不写，则会删除所有数据-通常都不会这么用
修改数据 update 表名 set 字段名1=新值1, 字段名2=新值2 [where 条件]; 查询数据 select 字段列表 from 表名 where 条件
